Terms & Acronyms

A comprehensive glossary of networking, cybersecurity, and computing terms used throughout this site.

Term / Acronym Description
DNS (Domain Name System) A hierarchical naming system that translates human-readable domain names into IP addresses.
IP (Internet Protocol) A protocol that provides addressing and routing for data packets across networks.
TCP (Transmission Control Protocol) A reliable, connection-oriented protocol that ensures data is delivered in order and without errors.
UDP (User Datagram Protocol) A connectionless protocol that provides fast but unreliable data transmission without error checking.
HTTP (Hypertext Transfer Protocol) The protocol used for transmitting web pages and data between browsers and servers.
HTTPS (HTTP Secure) HTTP encrypted with TLS/SSL to provide secure communication over the internet.
VPN (Virtual Private Network) A secure tunnel that encrypts internet traffic and masks your IP address for privacy and security.
ISP (Internet Service Provider) A company that provides internet access and routing services to end users.
TTL (Time To Live) The amount of time a DNS record or packet is cached before it must be refreshed.
OSI Model A 7-layer conceptual framework that standardizes the functions of a telecommunication system.
Root Server One of 13 authoritative DNS servers at the top of the DNS hierarchy that knows where TLD servers are.
TLD (Top-Level Domain) The last part of a domain name, such as .com, .org, or .net, managed by TLD nameservers.
Authoritative Server The DNS server that holds the definitive DNS records for a specific domain.
Resolver A DNS server that performs recursive lookups on behalf of clients to find IP addresses for domain names.
Cache Temporary storage of DNS records or web content to speed up subsequent requests.
GET An HTTP method used to retrieve data from a server without modifying it.
POST An HTTP method used to submit data to a server, typically for creating or updating resources.
PUT An HTTP method used to update or replace an entire resource on the server.
DELETE An HTTP method used to remove a resource from the server.
Status Code A three-digit number in HTTP responses indicating the result of a request (e.g., 200 OK, 404 Not Found).
Header Metadata sent with HTTP requests and responses that provide additional information about the message.
Cookie Small pieces of data stored by browsers and sent back to servers to maintain state between requests.
TLS (Transport Layer Security) A cryptographic protocol that provides secure communication over networks, successor to SSL.
SSL (Secure Sockets Layer) An older cryptographic protocol for secure communication, now largely replaced by TLS.
Encryption The process of converting plaintext into ciphertext to protect data from unauthorized access.
Authentication The process of verifying the identity of a user, device, or system.
ls Linux command to list files and directories in the current location.
cd Linux command to change the current directory.
grep Linux command to search for patterns in text files or output.
Package Manager Software tool for installing, updating, and removing software packages on a system (e.g., apt, yum).
Log Records of system events, errors, and activities stored in files for troubleshooting and monitoring.
Permissions Access control settings that determine what actions users can perform on files and directories.
Port A numbered endpoint used to identify specific services or applications on a network device.
Firewall A network security device that monitors and controls incoming and outgoing network traffic.
Packet A small unit of data transmitted over a network, containing headers and payload information.
Router A network device that forwards data packets between different networks based on IP addresses.
Switch A network device that connects devices on a local network and forwards data based on MAC addresses.
MAC Address A unique identifier assigned to network interfaces for communications at the data link layer.
Subnet A logical subdivision of an IP network that allows for better organization and routing.
DHCP (Dynamic Host Configuration Protocol) A protocol that automatically assigns IP addresses and network configuration to devices on a network.
ARP (Address Resolution Protocol) A protocol that maps IP addresses to MAC addresses on a local network.
ICMP (Internet Control Message Protocol) A protocol used for network diagnostics and error reporting, commonly used by ping and traceroute.
NAT (Network Address Translation) A technique that allows multiple devices on a private network to share a single public IP address.
Proxy An intermediary server that acts on behalf of clients, forwarding requests and responses.
SSH (Secure Shell) A cryptographic network protocol for secure remote login and command execution over an unsecured network.
FTP (File Transfer Protocol) A standard network protocol used for transferring files between a client and server on a network.
API (Application Programming Interface) A set of protocols and tools that allows different software applications to communicate with each other.
JSON (JavaScript Object Notation) A lightweight data interchange format that is easy for humans to read and write, and for machines to parse.
Load Balancer A device or software that distributes network traffic across multiple servers to improve performance and reliability.
