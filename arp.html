ARP — Address Resolution Protocol

Translating IP addresses to MAC addresses for local delivery

When your device knows an IP address but needs to physically send data on the local network, ARP finds the MAC address.

❓ The Problem
  • You want to send data to 192.168.1.50
  • Your network card only understands MAC addresses
  • How do you find the MAC?
✓ The Solution
  • ARP broadcasts: "Who has 192.168.1.50?"
  • That device responds: "Me — here's my MAC"
  • Now you can send directly

How ARP Works

1
ARP Request Broadcast

Your device sends a message to everyone on the local network asking who owns the target IP.

Your MAC → ff:ff:ff:ff:ff:ff
"Who has 192.168.1.50? Tell 192.168.1.10"
2
ARP Reply Unicast

Only the device with that IP responds, sending its MAC address directly back to you.

aa:bb:cc:dd:ee:ff → Your MAC
"192.168.1.50 is at aa:bb:cc:dd:ee:ff"
💾
Result Cached
The mapping is stored in your ARP table so future packets don't need another lookup.
192.168.1.50 → aa:bb:cc:dd:ee:ff

Key Concepts

Concept Explanation
ARP Request Broadcast asking "who has this IP?"
ARP Reply Response containing the MAC address
ARP Cache Local table storing IP-to-MAC mappings
Cache Expiry Entries expire after minutes to stay fresh
Broadcast MAC ff:ff:ff:ff:ff:ff — "send to everyone"
Layer 3 — IP
ARP
Layer 2 — MAC
Where ARP Fits
  • Layer 2 protocol (works with MAC addresses)
  • Bridges Layer 3 (IP) to Layer 2 (MAC)
  • Only used for local network communication

When ARP Happens

Scenario ARP Used For
Sending to local device Find that device's MAC
Sending to remote address Find your router's MAC
Receiving from internet Router finds your MAC

Commands Reference

OS Command What It Shows
Linux / Mac arp -a Your ARP cache
Windows arp -a Your ARP cache
Red Team Relevance

ARP has no authentication. Any device can respond to an ARP request — even if it's lying. This enables ARP Spoofing: an attacker responds to ARP requests with their own MAC, redirecting traffic through their machine. This is a common technique for man-in-the-middle attacks on local networks.
