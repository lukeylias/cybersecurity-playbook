Wireshark Basics

See what's actually happening on the network

Wireshark captures and displays network traffic — every packet, every conversation, every piece of data traveling across the wire.

What is Wireshark?

Wireshark is a packet analyzer. It captures network traffic and lets you inspect every packet — what was sent, where it went, what protocol was used, and what data it contained.

Need CLI-based capture? If you're on a remote server via SSH or a compromised box without a GUI, use tcpdump to capture traffic, then transfer the .pcap file here for analysis.

Why It Matters

Use Case What You Can Do
Learning See how protocols actually work (TCP handshakes, HTTP requests, DNS lookups)
Troubleshooting Find why a connection is failing, identify network bottlenecks
Security Analysis Spot suspicious traffic, analyze malware communication patterns
CTFs Extract files from captures, find hidden data, decrypt traffic with keys

Essential Navigation

Quick reference for navigating captures efficiently.

Task How to Do It
Go to specific packet Ctrl + G, type packet number
Find text in packets Ctrl + F, select search options
Apply a display filter Type in filter bar at top, press Enter
Clear filter Click X in filter bar or clear the text
Follow a conversation Right-click packet → Follow → TCP/HTTP/UDP Stream
Mark a packet Ctrl + M to toggle mark on selected packet
Jump to next marked Ctrl + Shift + N

Common Filters

Filters let you narrow down to just the traffic you care about.

Filter What It Shows
http Only HTTP traffic
dns Only DNS queries and responses
tcp.port == 443 Only traffic on port 443 (HTTPS)
ip.addr == 192.168.1.5 Traffic to or from that IP
ip.src == 192.168.1.5 Traffic from that IP (source)
ip.dst == 192.168.1.5 Traffic to that IP (destination)
tcp.flags.syn == 1 SYN packets (connection starts)
http.request.method == "GET" Only HTTP GET requests
http.request.method == "POST" Only HTTP POST requests
frame contains "password" Packets containing the word "password"
Tip: Filters are case-sensitive. You can combine them with && (and) or || (or).

Example: ip.addr == 192.168.1.5 && http shows only HTTP traffic involving that IP.

Common Tasks

Step-by-step guides for things you'll do frequently in Wireshark.

📁
Extract a File from Captured Traffic
When you see a file was downloaded and want to save it
EXPAND
Go to File menu → Export Objects → HTTP (or SMB, IMF, etc.)
A window appears showing all transferred files in the capture
Find the file you want in the list
Select it → Click Save
Open the saved file on your machine
CTF Tip: Flags are often hidden in images, PDFs, or zip files transferred over HTTP. Always check Export Objects!
💬
Follow a Conversation
When you want to see the full back-and-forth between client and server
EXPAND
Find a packet from the conversation you're interested in
Right-click the packet
Select Follow → TCP Stream (or HTTP Stream, UDP Stream)
A window shows the complete conversation — requests and responses
Note: Different colors show different directions — typically red for client and blue for server (or vice versa).
📝
View Packet Comments
CTFs sometimes hide hints in packet comments
EXPAND
Select the packet in the packet list
Look in the Packet Details pane for "Packet comments" or a comment icon
Or right-click → Packet Comment → View Comment
Or go to Edit menu → Packet Comment
CTF Tip: Filter for packets with comments using pkt_comment
🔓
Decrypt HTTPS/TLS Traffic
When you have the TLS session keys and want to read encrypted traffic
EXPAND
Go to Edit → Preferences
Expand Protocols in the left sidebar → find TLS (or SSL in older versions)
Find "(Pre)-Master-Secret log filename"
Browse to your key log file (often named sslkeylog.txt)
Click OK — traffic should now be decrypted
Important: You can only decrypt traffic if you have the session keys. Browsers can be configured to dump these keys to a file by setting the SSLKEYLOGFILE environment variable.
🔢
Find the Number of Filtered Packets
When a question asks "how many packets match X criteria?"
EXPAND
Apply your filter in the filter bar
Look at the bottom right of the Wireshark window
You'll see "Displayed: X of Y" — X is your filtered count
Example: With filter http.request.method == "GET", the status bar might show "Displayed: 47 of 1,234" meaning 47 GET requests in the capture.

Understanding What You See

Packet Colors

Wireshark color-codes packets by type. Here are the default colors:

Light Purple
TCP traffic
Light Blue
UDP traffic
Light Green
HTTP traffic
Black/Red
Errors, problems
Light Yellow
SMB/Windows traffic

Protocol Layers

When you expand a packet in the details pane, you see layers (like the OSI model):

Frame Physical capture info (timestamps, length) Physical
Ethernet MAC addresses Layer 2
IP IP addresses, TTL, protocol Layer 3
TCP / UDP Source and destination ports, flags, sequence numbers Layer 4
HTTP / FTP / etc. Application-specific data and headers Layer 7

Red Team Relevance

🎯 Security Analysis Use Cases
Use Case What to Look For
Credential Capture Unencrypted logins (HTTP, FTP, Telnet) — search for password or login
Data Exfiltration Large outbound transfers, unusual destinations, DNS tunneling
Recon Detection Port scans (many SYN packets to different ports), unusual traffic patterns
Malware Analysis Command and control traffic, beacons, encoded/base64 data in HTTP
Man-in-the-Middle ARP spoofing evidence, certificate anomalies in TLS handshakes

Glossary

Packet
A single unit of network data with headers and payload.
Capture
The act of recording network traffic, or the resulting file.
Filter
A rule to show only packets matching certain criteria.
Stream
A complete conversation between two endpoints.
Pcap / Pcapng
File formats for saved packet captures. Pcapng is newer with more features.
Display Filter
Filters what you see in the UI (doesn't affect what's captured).
Capture Filter
Filters what gets captured (set before starting capture).
Follow Stream
View the complete back-and-forth of a TCP/HTTP/UDP conversation.
Export Objects
Extract files (images, documents, etc.) transferred in the capture.
Dissector
Wireshark's protocol decoder that parses packet data into readable fields.
