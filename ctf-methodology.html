PHASE 1
Scan ‚Äî Find the Doors
"You cannot attack what you cannot see"
COLLAPSE
Primary Scan Command
nmap -sC -sV -oN nmap_initial.txt <target-ip>
COPY
-sC Run default scripts ‚Äî checks for common vulnerabilities and gathers extra info
-sV Detect service versions ‚Äî crucial for finding exploits (e.g., Apache 2.4.49)
-oN Save output to file ‚Äî always document your findings
Additional Scans
Situation Command When to Use
Quick scan nmap -F <ip> Initial fast look (top 100 ports)
All ports nmap -p- <ip> When default scan finds little ‚Äî checks all 65535
UDP scan nmap -sU --top-ports 20 <ip> When TCP shows nothing interesting
Aggressive nmap -A <ip> OS detection + traceroute + scripts (noisy)
üìù
Document Everything
Write down every open port and version number. Each version is a potential key ‚Äî search for exploits using the exact version string.

Web Enumeration
Port 80 (HTTP) / Port 443 (HTTPS)
EXPAND
Goal

Find hidden pages, admin panels, vulnerable applications, and version numbers to exploit.

Step Action Command Why
1 Browse the site Open in browser Visual inspection, read comments, look for names/usernames
2 Check robots.txt curl http://<ip>/robots.txt Lists hidden paths the site doesn't want indexed
3 Find hidden directories gobuster dir -u http://<ip> -w /usr/share/wordlists/dirb/common.txt Find /admin, /backup, /uploads, etc.
4 Identify technology whatweb http://<ip> Identifies CMS (WordPress, Joomla), frameworks, versions
5 Vulnerability scan nikto -h http://<ip> Automated check for misconfigurations
6 Check source code Right-click ‚Üí View Source Look for comments, hidden forms, JavaScript clues
Common Findings ‚Üí Next Steps
Found login page ‚Üí Try default creds (admin/admin), then consider brute force with hydra
Found WordPress ‚Üí Run wpscan --url http://<ip> -e ap,at,u
Found file upload ‚Üí Test if you can upload a PHP shell (.php, .phtml, .php5)
Found version number ‚Üí Search: searchsploit <software> <version>

FTP Enumeration
Port 21
EXPAND
Goal

Check for anonymous access, find sensitive files, identify vulnerable versions.

Step Action Command Why
1 Try anonymous login ftp <ip> ‚Üí username: anonymous Common misconfiguration allows guest access
2 List all files ls -la Look for configs, passwords, scripts, hidden files
3 Download files get filename Examine files locally for credentials or clues
4 Check for upload put testfile.txt If writable, you might upload a shell
5 Check version searchsploit <ftp-version> e.g., vsftpd 2.3.4 has a famous backdoor
üí°
Pro Tip
If anonymous login works, always check if FTP is connected to the web root. Files you upload via FTP might be accessible via HTTP.

SMB Enumeration
Port 445 (also 139)
EXPAND
Goal

Find open shares, enumerate users, discover sensitive files.

Step Action Command Why
1 List shares smbclient -L //<ip> See what shares exist on the target
2 Try null session smbclient //<ip>/sharename -N -N means no password (anonymous)
3 Full enumeration enum4linux -a <ip> Extracts users, groups, shares, password policy
4 Connect to share smbclient //<ip>/share -U user If you found credentials, access the share
Once connected to a share
ls # List files
cd directory # Change directory
get filename # Download file
mget * # Download all files

SSH
Port 22
EXPAND
‚ö†Ô∏è
Important
SSH is rarely the first entry point. Usually you need credentials found elsewhere (web, FTP, SMB) before SSH becomes useful.
Step Action Command Why
1 Try found credentials ssh user@<ip> If you found creds from web/ftp/smb
2 Use private key ssh -i id_rsa user@<ip> If you found an SSH private key file
3 Brute force (last resort) hydra -l user -P /usr/share/wordlists/rockyou.txt ssh://<ip> Only if you have a confirmed username
üí°
Found a Private Key?
Set correct permissions first: chmod 600 id_rsa then connect with ssh -i id_rsa user@ip

PHASE 3
Exploit ‚Äî Get In
"Use what you found to gain access"
COLLAPSE
The Systematic Approach
Step Action Details
1 Search for exploits searchsploit <service> <version> or Google: "<service> <version> exploit CVE"
2 Read the exploit Understand what it does. Check if it needs modifications (IP, port, path)
3 Start your listener nc -lvnp 4444 ‚Äî Always start BEFORE running exploit
4 Run the exploit Execute and watch your listener for incoming connection
Start Your Listener First
nc -lvnp 4444
COPY

Run this BEFORE executing any reverse shell exploit. It waits for the target to connect back to you.

If You're Stuck

Go back to basics. The answer is usually in something you missed, not something advanced.

‚Üª Restart Checklist
Did I scan ALL ports? (nmap -p-)
Did I check UDP ports?
Did I Google every version number?
Did I check page source code on all web pages?
Did I try default credentials? (admin/admin, root/root)
Did I check anonymous access? (FTP, SMB)
Did I read ALL files I found?
Did I try a bigger wordlist for gobuster?
Did I run LinPEAS/WinPEAS?
Did I check sudo -l?

Glossary

Term What It Is
Reverse Shell A connection from the target back to your machine, giving you command-line access
SUID File permission that runs the program as its owner (often root), regardless of who executes it
Privilege Escalation Moving from a low-level user account to root/administrator level access
Banner Grab Connecting to a service to see what information it reveals (version, software name)
Enumeration Gathering detailed information about discovered services, users, and configurations
CVE Common Vulnerabilities and Exposures ‚Äî a standardized ID for known security flaws (e.g., CVE-2021-44228)
RCE Remote Code Execution ‚Äî ability to run commands on a target system remotely
LinPEAS / WinPEAS Automated scripts that search for privilege escalation vectors on Linux/Windows
GTFOBins Website listing Unix binaries that can be exploited for privilege escalation
Payload The code or command that performs the actual exploit action (e.g., spawns a shell)
Listener A program waiting for incoming connections (like nc -lvnp 4444)
