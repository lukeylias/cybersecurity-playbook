Protocols — How Services Communicate

What they are, which port they use, and how to connect

The Key Concept

When you scan a target and find open ports, each port tells you what service is running and how to talk to it.

Protocol Reference

80/443
HTTP / HTTPS
View and interact with websites
Protocol Port Encryption
HTTP 80 Not encrypted
HTTPS 443 Encrypted (TLS)
How to Connect
Browser
http://10.10.10.5
Terminal
curl http://10.10.10.5
You don't need to add :80 — browsers assume it for http://
⚙ Red Team Tips
  • Run gobuster dir -u http://IP -w wordlist for hidden directories
  • Check page source for comments and hidden info
  • Look for login pages, admin panels, backup files
21
FTP
Transfer files to/from a server
How to Connect
Connect to FTP
ftp 10.10.10.5
Then enter credentials
Username: anonymous
You →
USER anonymous
← Server
331 Password required
You →
PASS email@example.com
← Server
230 Login successful
You →
LIST
Server shows available files
Command What it does
ls List files
get file.txt Download file
put file.txt Upload file
quit Disconnect
⚙ Red Team Tips
  • Try anonymous login first (user: anonymous, pass: anything)
  • Look for config files, backups, credentials
  • Check if you can upload files (potential webshell)
22
SSH
Secure remote command-line access
How to Connect
With password
ssh username@10.10.10.5
With SSH key
ssh -i key.pem user@10.10.10.5
You need valid credentials — username plus password or SSH key
⚙ Red Team Tips
  • Banner grab for version: nc 10.10.10.5 22
  • Brute force with hydra if you have usernames
  • Check for key-based auth, look for leaked private keys
25
SMTP
Sending emails
How to Connect
Manual connection
telnet 10.10.10.5 25
Or with netcat
nc 10.10.10.5 25
You →
HELO mydomain.com
You →
MAIL FROM: me@example.com
You →
RCPT TO: target@example.com
You →
DATA
Then type message, end with single period (.)
⚙ Red Team Tips
  • Check for open relay (can you send to external addresses?)
  • User enumeration with VRFY username command
  • Try EXPN to expand mailing lists
53UDP/TCP
DNS
Translate domain names to IP addresses
How to Use
nslookup
nslookup example.com
dig (more detailed)
dig example.com
⚙ Red Team Tips
  • Zone transfer attempt: dig axfr @nameserver domain.com
  • Subdomain enumeration with wordlists
  • Reverse DNS lookups on IP ranges
110/143
POP3 / IMAP
Receiving and reading emails
Protocol Port Behavior Best for
POP3 110 Downloads & deletes from server Single device
IMAP 143 Syncs, stays on server Multiple devices
How to Connect
POP3
telnet 10.10.10.5 110
IMAP
telnet 10.10.10.5 143
⚙ Red Team Tips
  • Try default/common credentials
  • Look for emails containing passwords or sensitive info
  • Check secure versions (POP3S: 995, IMAPS: 993)

Quick Reference

Protocol Port What you do Command to connect
HTTP 80 Browse website Browser or curl http://IP
HTTPS 443 Browse secure site Browser or curl https://IP
FTP 21 Transfer files ftp IP
SSH 22 Remote shell ssh user@IP
SMTP 25 Send email telnet IP 25
DNS 53 Lookup domains nslookup or dig
POP3 110 Receive email telnet IP 110
IMAP 143 Sync email telnet IP 143
RDP 3389 Windows remote desktop rdesktop IP
SMB 445 Windows file shares smbclient -L IP
Glossary — Key terms used on this page
Term What it is
Banner grab Connecting to a port to see what info the service reveals (version, software name)
Open relay An SMTP server that allows anyone to send email through it — often exploited for spam
Zone transfer Copying all DNS records from a server — often misconfigured to allow anyone to dump the zone
Anonymous login FTP servers that allow login without real credentials (user: anonymous)
TLS/SSL Encryption protocols that secure data in transit — the "S" in HTTPS
Port A numbered endpoint (0-65535) identifying a specific service on a machine
